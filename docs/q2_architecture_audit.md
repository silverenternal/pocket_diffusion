# Q2 Architecture Audit

This audit freezes the implemented Q2 baseline and labels later config-selectable flow-head ablations without promoting them as the default.

The default flow system is `GeometryFlowMatchingHead`. It is a preserved baseline, not a missing feature. It predicts per-atom Cartesian velocity from centered coordinates, initial coordinates, displacement, timestep features, six directed gate summaries, and mean-pooled topology, geometry, and pocket contexts.

The current flow is geometry-first. It is not a full molecular flow: atom types, topology, and bond structure are not jointly generated by the flow objective, and the velocity head outputs only `[num_atoms, 3]` coordinate velocities.

## Capability Status

| Capability | Status | Claim boundary |
| --- | --- | --- |
| MLP fusion velocity head | implemented | Claim as active baseline architecture. |
| Mean-pooled modality conditioning | implemented | Claim as current behavior and known compression risk. |
| Local atom-pocket attention in velocity head | config-selectable ablation | `AtomPocketCrossAttentionVelocityHead` is selected by `model.flow_velocity_head.kind = "atom_pocket_cross_attention"`; it is not the default. |
| Ligand pairwise message passing in flow head | config-selectable ablation | `model.pairwise_geometry.enabled` injects bounded pairwise geometry messages into the configured velocity head; it is off by default. |
| Rotation augmentation | implemented | Config-controlled same-rotation augmentation is wired through manifest loading and reports attempted/applied counts; flow-head smoke coverage still treats rotation consistency as a diagnostic rather than a full equivariance claim. |
| Topology flow | absent | Do not claim full molecular or topology flow. |
| Bond logits in flow objective | absent | Bond handling is downstream/refinement/postprocessing-dependent. |
| Hard pocket-envelope constraints | implemented | Must be reported separately from raw model-native output. |
| Layer attribution | implemented with Q2 contract | Use `raw_flow`, `constrained_flow`, `repaired`, and `reranked` roles explicitly. |

## Claim Rules

Q2 architecture claims should call the default baseline "geometry-first flow matching" and should not imply full molecule generation through flow. Config-selected local pocket attention or pairwise geometry should be discussed as ablation capacity, not as the promoted default. Any quality attributed to hard constraints or repair must be separated from raw velocity-field capability.
