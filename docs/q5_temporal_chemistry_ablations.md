# Q5 Temporal Sync And Chemistry Collaboration Ablations

These smoke configs isolate the Q5 synchronization and chemistry-aware collaboration controls while keeping the executable path synthetic, CPU-only, and backend-free. They are intended for config validation, smoke instantiation, and regression gating before scaling the same variants to real held-out pocket splits.

All configs use `conditioned_denoising`, `candidate_count=1`, `enable_comparison_runner=false`, synthetic data, and disabled external chemistry/docking/pocket backends. Any pharmacology-facing backend or experimental evidence should therefore remain unavailable in claim artifacts.

| Config | Isolated mechanism | Expected metric movement | Failure mode to watch |
| --- | --- | --- | --- |
| `configs/q5_no_temporal_policy_smoke.json` | Refreshes rollout context every step but removes stage, rollout, and flow-time interaction multipliers. | Synchronization counters should remain clean, while stage/step-specific gate usage should flatten relative to the full variant. | Good task metrics with flat path timing would indicate that temporal policy is not carrying measurable signal. |
| `configs/q5_static_context_only_smoke.json` | Keeps temporal path policy but freezes inference context refresh. | `refresh_count` should fall and `stale_context_steps` can increase during rollout; chemistry metrics should not be promoted to stronger evidence. | Strong rollout metrics despite stale context need review for leaked or cached pocket state. |
| `configs/q5_no_chemistry_bias_smoke.json` | Keeps directed interaction and temporal scheduling but removes ligand-pocket distance and chemistry-role attention bias plus chemistry auxiliary weights. | Chemistry role coverage and guardrail improvements should weaken; gate usage may remain active but less chemically structured. | If pharmacophore coverage improves, check whether diagnostics are reporting heuristic availability rather than a real collaboration change. |
| `configs/q5_distance_bias_only_smoke.json` | Keeps ligand-pocket distance bias and geometric pocket losses while disabling chemistry-role bias and pharmacophore/valence/bond guardrails. | Clash/contact geometry can improve, while pharmacophore coverage and topology chemistry guardrails should lag the full variant. | Valence or pharmacophore gains here would suggest non-isolated chemistry signals or mislabeled provenance. |
| `configs/q5_no_topology_pocket_pharmacophore_smoke.json` | Keeps geometry-pocket chemistry bias but disables the two topology-pocket directed paths. | Topology-pocket pharmacophore diagnostics and role-aware gate usage should drop relative to full collaboration. | If topology-pocket coverage remains high, inspect disabled-path handling and diagnostics derived from zeroed attention. |
| `configs/q5_full_sync_chemistry_collaboration_smoke.json` | Enables every-step refresh, temporal path policy, distance plus role bias, pharmacophore probes, leakage probes, and chemistry guardrails. | Clean synchronization, nonzero refresh counts, interpretable path-role gate usage, and available heuristic chemistry collaboration metrics. | Any external-evidence claim, nonzero sync mismatch, or unavailable core heuristic diagnostic should block scaling this surface. |

Scale-up runs should preserve the same variant labels and only change dataset size, backend availability, and training budget. Backend-supported, docking-supported, and experimental evidence must stay provenance-labeled rather than replacing these heuristic smoke metrics.
