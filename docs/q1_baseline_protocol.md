# Q1 Baseline Protocol

Public baselines are registered in `configs/q1_baseline_registry.json`. Each baseline must preserve its raw generated candidates as `raw_rollout` and then pass through the same candidate metric pipeline used by the main method.

Current public-baseline status:

- Pocket2Mol, TargetDiff, and DiffSBDD are adapted on the fixed 100-pocket public-test set.
- The claim-facing layer-separated output is `checkpoints/q1_public_baselines_full100_layered/merged/candidate_metrics_q1_public_full100_budget1.jsonl`.
- `raw_rollout` rows are native public-baseline evidence.
- `repaired` and `reranked` rows are generated by `tools/public_baseline_postprocess_layers.py` and are postprocessing evidence.
- Current 900-row coverage is RDKit 1.0, GNINA 1.0, and Vina 0.9656.

Required comparison rules:

- identical unseen-pocket split
- matched candidate count per pocket
- recorded sampling steps or explicit `not_applicable`
- same RDKit, Vina, GNINA, pocket-contact, interaction, scaffold, and geometry backends
- separate raw, repaired, and reranked layers
- unavailable public baselines remain in the registry with a blocker reason

The generic adapter entry point is `tools/baseline_output_adapters/generic_generation_layers_adapter.py`. It converts JSON, JSONL, or CSV candidate exports into `generation_layers_<split>.json`; candidate metrics are then built with `tools/build_candidate_metrics_from_generation_artifacts.py`. The current public-baseline layer-separated run additionally uses `tools/public_baseline_postprocess_layers.py` before docking-input preparation and Vina/GNINA rescoring.

Public baseline acquisition sources:

| baseline | source | run output expected by this repo |
| --- | --- | --- |
| Pocket2Mol | `https://github.com/pengxingang/Pocket2Mol` | generated candidates adapted to `generation_layers_test.json` |
| TargetDiff | `https://github.com/guanjq/targetdiff` | generated candidates or official sampling meta converted to candidate rows |
| DiffSBDD | `https://github.com/arneschneuing/DiffSBDD` plus Zenodo checkpoint `https://zenodo.org/record/8183747/files/crossdocked_fullatom_cond.ckpt` | generated SDF candidates adapted to `generation_layers_test.json` |

These sources identify where to obtain runnable baseline code or checkpoints; they are not themselves claim-bearing evidence until the outputs are generated on the fixed benchmark split and rescored by this repository's RDKit, Vina, GNINA, pocket-contact, scaffold, geometry, and interaction metric pipeline.
